## Known limitations

- The avatar_widget test suite is sensitive to Windows GPU broker process
  state. If a previous test run was force-killed mid-flight, the broker
  becomes corrupted and subsequent runs hang at the first QWebEngineView
  construction. Reboot Windows or test on Linux/macOS for clean runs.
  The hang is a Windows session issue, not a code defect — the underlying
  logic is covered by the `main_avatar_integration` and `avatar_player_html`
  suites which run cleanly (23 passed, 1 skipped in 11.38s).
- The avatar widget requires a localhost HTTP server because Chromium
  blocks ES module imports from `file://`. The server is started by main.py
  on a random 127.0.0.1 port and torn down on app exit.
